# Report Structure Reference

## Report Architecture

### Standard Thematic Research Report Outline

```
1. Executive Summary (1-2 pages)
2. Theme Deep Dive (5-10 pages)
   2.1 Theme Definition & Drivers
   2.2 Market Sizing & TAM
   2.3 Value Chain Analysis
   2.4 Competitive Landscape
   2.5 Regulatory Environment
   2.6 Theme Timeline & Catalysts
3. Investment Universe (2-3 pages)
   3.1 Universe Construction
   3.2 Screening Results
   3.3 Peer Groups
4. Company Profiles (3-5 pages each)
   4.1 Company Overview
   4.2 Thematic Positioning
   4.3 Financial Analysis
   4.4 Valuation
   4.5 Risks & Catalysts
   4.6 Investment Recommendation
5. Risk Assessment (2-3 pages)
   5.1 Theme-Level Risks
   5.2 Company-Level Risks
   5.3 Thesis Killers
   5.4 Mitigation Strategies
6. Implementation Guide (1-2 pages)
   6.1 Portfolio Construction
   6.2 Entry Timing
   6.3 Monitoring Framework
   6.4 Exit Criteria
7. Appendices
   A. Detailed Financial Models
   B. Full Universe List
   C. Source Documentation
   D. Glossary
   E. Disclosures
```

## Quality Standards

### Analytical Rigor Checklist

| Standard | Met? | Evidence |
|----------|------|----------|
| All quantitative claims sourced | | |
| Assumptions explicit and justified | | |
| Sensitivity analysis included | | |
| Multiple valuation methods used | | |
| Competing hypotheses addressed | | |
| Logic chain explicit | | |
| Data sources documented | | |

### Presentation Standards

| Standard | Requirement |
|----------|-------------|
| Executive Summary | Standalone decision-quality |
| Visual Elements | Charts for data-dense content |
| Formatting | Consistent throughout |
| Terminology | Defined in glossary |
| Length | Appropriate to complexity |

### Common Quality Issues to Avoid

| Issue | Problem | Solution |
|-------|---------|----------|
| Confirmation bias | Only supporting evidence | Include bear case |
| Unsourced claims | Can't be verified | Cite all data |
| Circular logic | Thesis proves itself | External validation |
| Overconfidence | No uncertainty acknowledged | Scenario ranges |
| Stale data | Outdated information | Check recency |

## Report Review Process

### Pre-Publication Checklist

**Content Quality:**
- [ ] Thesis clearly stated
- [ ] All claims sourced
- [ ] Assumptions explicit
- [ ] Sensitivity analysis complete
- [ ] Risks comprehensively addressed
- [ ] Recommendations actionable

**Technical Quality:**
- [ ] Calculations verified
- [ ] Models audited
- [ ] Data current
- [ ] Cross-references accurate

**Presentation Quality:**
- [ ] Executive summary standalone
- [ ] Logical flow
- [ ] Consistent formatting
- [ ] Proofread for errors
- [ ] Charts clear and labeled

**Compliance:**
- [ ] Disclosures included
- [ ] Conflicts documented
- [ ] Appropriate caveats
